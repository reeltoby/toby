Methodology: What was done

PREPARING THE STUDY AREA

Weather Stations:

Downloaded the entire weather stations data of Canada from Google as a KML file (https://www.google.com/maps/d/viewer?msa=0&dg=feature&mid=1zXdPVrRpsXd37wTN3nzcF fCQgdE&ll=47.12315462087478%2C-78.23151418166714&z=5) then exported our study area as a layer on its own. This Layer was then brought to the map and clipped with the weather stations data to extract the weather stations in Ontario.

Weather stations all around Canada

Clipped Weather stations in Ontario

Weather Data Download:

We automated the download of weather data according to weather stations in Ontario, extracting the data and stored them in PostGIS postgresql databse. This data contains other information and details like precipitation, humidity, but our main focus is on wind speed and temperature. The information automatically saved in the database has temperature and wind speed record for all the weather stations in Ontario. The database was then connected to QGIS and the data brought over as a layer for spatial analysis to decipher the best interpolation methods.

SPATIAL ANALYSIS

We created two CSV files holding the latitude, longitude, and station codes of weather stations within our study area. These files would then be used in the ETL automation script. The script downloads hourly weather station data from this Government of Canada website: 'https://dd.weather.gc.ca/nowcasting/matrices/' using the python URLLIB module, unzips the downloaded file, and converts it to a text file.

PYQGIS

PyQGIS is a framework that allows you to run QGIS processes in python. To run QGIS processes in python, we needed to set our python interpreter to the python-qgis.bat interpreter. The pyQGIS would not connect to postgresql database to pull data for interpolation as we tried several methods and sough online helps.

BATCH FILE

We set up a batchfile to automatically run python scripts and download the current weather data from the government of Canada website. This would the carry out the interpolation and store the data in the server We faced challenges with this and had to consult or client for possible solutions on how to go about this.

WEB DEVELOPMENT

The web application displays current weather information, wind speed, visibility and cloud condition. The web appliction is hosted on GitHub thus querying the web server provided by the client would be impossible. To remedy this, we retrieved weather data from openweathermap using the api provided by openweathermap.org. The Open weather Map is a featureopn-source weather resource and comes in as a valid replacement with high accuracy. The web apllication queries openweathermap.org using jQuery, supplying coordinates of the clicked points on the map, in turn retrieveing weather information for the clicked point. This process happens really fast. The map pane displays the user's current location onLoad, has a searchbar as well as a selection pane where map type, weather condition of interest could be chosen. These features are overlain on an Open Street Map base map which is a free and open source base map.
