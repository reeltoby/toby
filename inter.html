Methodology: What was done

PREPARING THE STUDY AREA

Weather Stations:

Downloaded the entire weather stations data of Canada from Google as a KML file then exported our study area as a layer on its own. This Layer was then brought to the map and clipped with the weather stations data to extract the weather stations in Ontario.

Weather stations all around Canada

Clipped Weather stations in Ontario

Weather Data Download:

We automated the download of weather data according to weather stations in Ontario, extracting the data and stored them in PostGIS postgresql databse. This data contains other information and details like precipitation, humidity, but our main focus is on wind speed and temperature. The information automatically saved in the database has temperature and wind speed record for all the weather stations in Ontario. A coonection was then established from QGIS destop application to the postgresql databse on our desktop and the data brought over as a layer for spatial analysis to decipher the best interpolation methods. This connection process led to installation of Geoserver(using apache tomcat) and postgresql in the web server hoping this would work seamlessly.

Overall, IDWP8 and Nearest neighbour seemed to be the most accurate. Nearest neighbour defaulted in its accuracy for unknown points as it just picks and displays the data from the nearest weather station. The results derived by the nearest neighbour interpolation method for unknown points were highly inaccurate. The IDW in its default stae was not as accurate as the when the Coefficient Distant parameter was set to 8. At the mode, we achieved the highest accuracy and agreed to go along with this interpolation method and embedded it into the python script.

PYQGIS

PyQGIS is a framework that allows you to run QGIS processes in python. To run QGIS processes in python, we needed to set our python interpreter to the python-qgis.bat interpreter. The pyQGIS would not connect to postgresql database to pull data for interpolation as we tried several methods and sough online helps.

BATCH FILE

We set up a batchfile to automatically run python scripts and download the current weather data from the government of Canada website. This would the carry out the interpolation and store the data in the server We faced challenges with this and had to consult or client for possible solutions on how to go about this.

WEB DEVELOPMENT

The web application displays current weather information, wind speed, visibility and cloud condition. The web appliction is hosted on GitHub thus querying the web server provided by the client would be impossible. To remedy this, we retrieved weather data from openweathermap using the api provided by openweathermap.org. The Open weather Map API is a featureopn-source weather resource and comes in as a valid replacement with high accuracy. The web apllication queries openweathermap.org using jQuery, supplying coordinates of the clicked points on the map, in turn retrieveing weather information for the clicked point. This process happens really fast. The map pane displays the user's current location onLoad, has a searchcbar as well as a selection pane where map type, weather condition of interest could be chosen. These features are overlain on an Open Street Map base map which is a free and open source base map. Leaflet javascript was made use of to build this map.
